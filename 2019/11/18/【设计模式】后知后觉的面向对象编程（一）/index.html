【设计模式】后知后觉的面向对象编程（一）

作为一个从JavaScript入门编程世界的人来说，所谓“面向对象”这个概念，就仿佛从来没真的存在过，它就像那传说中的大侠，江湖处处都有它的传说，却始终没曾有幸得识真面目。
然鹅，
路走多了就会想跑，摔倒了就会想怎么跳。

在自己的工作不再是简单的页面，而逐步进入越来越复杂的模块设计，尤其是Hybrid开发之后，误打误撞的，某些轮廓慢慢的浮现出来，究竟什么是类，什么是面向“对象”，在经历了一轮iOS（Objective-C）和Android（Java）的洗礼下，也逐渐开始明晰起来。
那种定义一个模型，并为其从简单到复杂，私有到公有，继承到多态的感觉，真好啊。

它似乎不再是机械式的通过一系列指令在与机器进行交谈，而是变得有轮廓，有毛发，有自己的呼吸的自生命体（只取决于设计人能力的强大与否）。

扯了这么多玄学，下面还是进入自我总结吧。

什么是面向对象编程

  1. 以类或对象作为组织代码的基本单元
  2. 将封装、抽象、继承、多态四个特性作为代码设计和实现的基石

什么是面向对象分析和面向对象设计

面向对象开发需要经历的三个阶段：面向对象分析（OOA）、面向对象设计（OOD）、面向对象编程（OOP）

  1. 始终围绕着类或对象来做需求设计、分析；
  2. 面向对象分析需要搞清楚要做什么；
  3. 面向对象设计来搞清楚怎么做；
  4. 而面向对象编程则是将需求和设计翻译成为代码的过程；
